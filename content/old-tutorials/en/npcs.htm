Tutorial
--------

Selfmade Non-Player-Characters ( NPCs) for Jedi-Knight II - Jedi Outcast.
=======================================================================================

to make the new NPC you need a NPC configuration file and of course the Glm-Model with all Skins and Sounds.

You can set the configuration in the "NPCS.CFG" or you can use a single ".npc"- file. Inside this file you can specify the properties abilities of your character.

The "npcs.cfg"-file exists in Folder "ext_data"of the assets0.pk3 – file in your JKII  /base -directory.
In this file are all configuration-data stored for all Single-Player-Charakters of JKII.

The possibility of using a single ".npc"-file is not documented anyway :) But you can use it to define a new NPC, without replacing an existing one and without editing of the "npcs.cfg". The file-structure of the ".npc"-file is equal to structure of "npcs.cfg", but can contain aas well only a single character definition. You can also have more than one ".npc"-file


Caution!:
_______
The ".npc"-file can not be used for replacing of existing characters (for example: if you want to replace Desann with Darth Vader). The only possibility to replace an existing charackter is to create new edited "npcs.cfg" ! Otherwise you will have problems with the sound and NPC-properties.


Jedi -NPC's, the bright side of the force
----------------------------------------
To make a good jedi, you must take a "jedi"-NPC as template.

All jedi-NPC must have names like this:

jedi_NPCname

where, "NPCname" is the name of your new NPC.

It's also important for the Model-folder! Otherwise it will not work!
Both, model- and NPC-names must begin with the word "jedi".


Example:
---------

Jedi_yoda
{
playermodel     jedi_yoda
saberColor       green
rank                 commander
reactions         5
aim                 5
move              5
aggression      5
evasion          5
intelligence     5
hfov                 180
vfov                 180
scale                 55
playerTeam     player
enemyTeam     enemy
// race human
class                 jedi
snd                 yoda
sndcombat     yoda
sndextra         yoda
sndjedi             yoda
yawSpeed         200
walkspeed         60
runspeed             220
health                 999
dismemberProbHead 0
dismemberProbArms 0
dismemberProbHands 0
dismemberProbLegs 0
dismemberProbWaist 0
}

rank - is skill-level of NPC (IQ, capacity of reaction ...)

There are following jedi rank-levels:

(only for good jedis)

captain - the best
Commander - very good
lt - good


raktion, aim, move, agression, evasion - properties of the abilities
1 = bad, 5 = very good

hfov,vfov - is visionradius (horizontal and vertical)

"scale" defines how big is the model in game, so is Desann bigger then yoda.


The following is also very important!:
-------------------------------------

good jedis must alse be defined like this:

PlayerTeam     player
EnemyTeam     enemy

"class " must  always be set to "jedi" !

if you change this, your new NPC will not properly work.

--------------------------------------
"race" is nonrelevant (Elite-Force)

"snd", "sndcombat" and so on, are sound-properties. Set it to "modelname"

"sound/chars/modelname/misc"

So if you save your new sounds in:

"sound/chars/Yoda"

it must be set as follows:

snd            Yoda
sndcombat Yoda
sndjedi       Yoda

----------------------------
yawSpeed is the rotate speed
walkspeed und runspeed are ... :)

"health" is the strengh of your NPC

IMPORTANT:
----------------

many of the mappers do not Know, that you can also define "halth" bigger than "999".

so, has "the Emperor" of my "The CHallenge..." -map abot 3000 health ! :)

ok, it was all about good jedis.

The Dark Side - Sith Warriors

The definition of dark jedis is similar to good jedis. There are only a few differences.

You must use the "Reborn"-NPC as template and replace the word "jedi" with "reborn".

it means:

NPCname         Reborn_Vader
Modelname         Reborn_Vader
class                 reborn

Playerteams must be set the other way around, as for  good-Jedi's. 

It means:

PlayerTeam     enemy
EnemyTeam     player


Also the rank levels are different:
---------------------------------
no rank defined - normal stupid reborn
crewman         - acrobat
ensign             - forceuser
ltjg                 - fencer
lt                     - boss
ltcomm            - shadowtrooper

commander    - very good (in game only set for Tavion)
captain            - the best (in the game only set for Desann)
-----------------------------------------------------------------
- Soldiers -

for enemy soldiers you must take "stromtrooper"-NPC as template and for allies
"BespinCop" or "rebel" - NPC. 

All other settings are analogical to "Jedi" and "reborn"-Settings

End-comments:
-----------------
It is recomended to make one ".npc"-file for all good guys and another one for the bad-guys.

If you have finished your NPC-denifintion, you can compress all the files in one ".pk3"-file

This file schould contain all the model/skin-files and folders as well sounds & shaders (optionally) you need for your NPC's.

It is improtant, that you use a relative-path in your pk3-file. It means, your folders should beginn after the /base - directory.

For examlpe, if you make a pk3-file for yoda-NPC, it should contain the following folders:

ext_data/ - here all ".npc"-files or one "npcs.cfg"-file

models/players/jedi_yoda/ - here all the model/skin -files

shaders/ - [otional] here all skin-shaders

sound/chars/jedi_yoda/misc/ - [optional] here all yoda sounds

 

If your "pk3." - file is done, you can copy it to your mod-folder or to your "base/" - folder (for testing only).
